# InkPulse 4.2 寸屏(SSD1677 BW)支持 — 设计

> 日期:2026-06-15 · 状态:已批准,待写实现计划

## 背景与目标

现有 InkPulse 使用 7.5 寸 **E075A42**(800×480,UC8179,BWR 三色)墨水屏。
现要新增对 4.2 寸 **YRD0426BBS770F0**(480×800,SSD1677 类,纯黑白 1-bit,支持局部刷新)的支持,
并满足三个目标:

1. **硬件**:核对新旧屏 FPC 是否 pin-to-pin 对应,判断是否改板。
2. **固件**:屏驱动充分解耦,能快速从 UC8179(BWR)切换到 SSD1677(BW)。
3. **hub 布局**:检测屏幕大小,基于现有 7.5 寸布局新实现一组 4.2 寸 480×800 竖屏布局。

一句话方案:**新屏与旧屏 pin-to-pin 兼容、不改板;固件编译期选驱动;hub 由设备上报的面板标识驱动多 profile 渲染(尺寸/颜色/旋转/网格/打包全随 profile)。**

## 已确认的硬事实(来自规格书)

新屏 `YRD0426BBS770F0`(`hardware/docs/YRD0426BBS770F0HP-V2.0-4.2_Inch-480x800.pdf`):

- 分辨率 **480(H)×800(V) 竖屏**;有效区 55.68×92.80mm;外形 62.37×105.33mm。
- **1-bit 纯黑白单 plane**(非 BWR);控制器 RAM 仍为 **800×480**,单 plane **48000 字节**(800/8×480)。
- 控制器命令集 `0x12/0x0C/0x01/0x11/0x44/0x45/0x4E/0x4F/0x24/0x22/0x20/0x10` =
  标准 **SSD16xx(SSD1677 类)**。
- 接口 4 线 SPI(BS1=L);信号:CS#/D-C#/RES#/SCL/SDA/BUSY;24-pin FPC;图像更新 ~1.1s;支持局部刷新;内置温度传感器。

> 注意:控制器 RAM 是 800×480(横向),模组按竖屏 480 宽×800 高观看。
> 因此"480×800 竖屏"在出帧时需要 **旋转 90°** 映射到 800×480 的 RAM。

## 数据流总览

```
设备(GET /frame?...&panel=bw_426)  ──►  hub
   │ 编译期 Kconfig 选屏                    │ panel id → ScreenProfile
   │ ssd1677 driver (display_if_t)          │ 选 layout 集 + 网格 + 颜色 + 旋转 + 打包
   ▼                                        ▼
 48000B 单plane ◄──── 800×480 1bpp ◄── rotate90 ◄── 渲染 480×800 竖屏画布
```

## 第 1 节 · 硬件(结论:**不改板**)

逐脚对比(旧屏 E075A42 规格书 p6 vs 新屏 YRD0426 规格书 p7):

| 脚 | 旧屏 E075A42(7.5″ BWR) | 新屏 YRD0426(4.2″ BW) | 一致 |
|---|---|---|---|
| 1 | NC | NC | ✓ |
| 2 | GDR | GDR | ✓ |
| 3 | RESE | RESE | ✓ |
| 4 | NC | NC | ✓ |
| 5 | VSHR | VSH2 | ✓ |
| **6** | **TSCL**(外置温感 I2C) | **NC** | ⚠ |
| **7** | **TSDA**(外置温感 I2C) | **NC** | ⚠ |
| 8 | BS | BS1 | ✓ |
| 9 | BUSY_N | BUSY | ✓ |
| 10 | RST_N | RES# | ✓ |
| 11 | DC | D/C# | ✓ |
| 12 | CS-N | CS# | ✓ |
| 13 | CLK/SCL | SCL | ✓ |
| 14 | DATA/SDA | SDA | ✓ |
| 15 | VDDIO | VDDIO | ✓ |
| 16 | VDDA | VCI | ✓ |
| 17 | VSS | VSS | ✓ |
| 18 | VDDD | VDD | ✓ |
| 19 | VPP | VPP | ✓ |
| 20 | VSH | VSH1 | ✓ |
| 21 | VGH | VGH | ✓ |
| 22 | VSL | VSL | ✓ |
| 23 | VGL | VGL | ✓ |
| 24 | VCOM | VCOM | ✓ |

**结论:24 脚中 22 脚完全同位,信号线(BS/BUSY/RST/DC/CS/SCL/SDA)全部一致 → 电气 pin-to-pin 兼容,不需要改板。**
唯一差异 pin6/7:旧屏是外置温感 I2C(TSCL/TSDA),新屏是 NC。本项目温湿度走独立板载 HTU21D、不经屏 FPC,故 NC 无影响。

交付物为一份**真机核对清单**(非代码):

1. 照新屏参考电路核对升压外围电容(GDR/RESE/VGH/VGL/VSH)值是否需微调。
2. 真机确认板上 pin6/7 未被强驱动到会干扰的电平(NC 一般安全)。
3. 新屏更小(62×105mm),**结构件/外壳/FPC 走线需重做**(属结构设计,不属改电路板)。

## 第 2 节 · 固件(编译期选驱动)

固件已有良好抽象:`ip_display/display.h` 定义 `display_if_t`(含 `get_caps`),`ip_channel` 的 `ch_init(caps)` 已用 `caps->frame_bytes` 决定拉多少字节。只需新增一个驱动实现 + 编译期选择。

- 新增组件驱动 `components/ip_display/src/ssd1677.c`,实现 `display_if_t`;
  `get_caps` 返回 `{width:480, height:800, color_model:DISP_BW, frame_format:"bw-1plane", frame_bytes:48000}`。
- `Kconfig` 增加 `choice INKPULSE_PANEL`:
  - `PANEL_UC8179_BWR_750`(默认,现状)
  - `PANEL_SSD1677_BW_426`
- `main.c` 按宏选驱动:
  ```c
  const display_if_t *disp =
  #if CONFIG_PANEL_SSD1677_BW_426
      ssd1677_driver();
  #else
      uc8179_driver();
  #endif
  ```
- 命令序列照规格书 p35 OTP 参考码:
  `0x12(软复位)→0x0C(升压软启动)→0x01(驱动输出/尺寸)→0x11(数据进入模式)→0x44/45(RAM XY 窗口)→0x4E/4F(RAM XY 计数器)→0x24(写 BW)→0x22(0xF7)→0x20(刷新)`;
  局刷走 SSD1677 partial update 流程(部分窗口 + partial LUT/0x22 参数)。
- `s_framebuf` 尺寸随 Kconfig(BW 仅需 48000;BWR 96000)。
- `mark_offline` 出 BW 变体:画黑色 X(无 red plane),与现 BWR 版同位。
- `ssd1677_selftest` 对标现有 `uc8179_selftest`:白/黑/棋盘 + 一次局部刷新演示。

**真机 bring-up 校准项**(非纯软件可定):

- 渲染→面板的旋转方向(90° 还是 270°)。
- 打包 bit 极性(`1`=黑 还是 `1`=白)。
- 升压软启动 / VCOM / LUT 参数微调。

URL 上报:固件在 `GET /frame` URL 追加 `&panel=<id>`(由 Kconfig 选屏决定,如 `bw_426` / `bwr_750`),供 hub 选 profile。

## 第 3 节 · hub(profile 驱动多尺寸)

### 现状耦合点

- `render/planes.py` 写死 `WIDTH,HEIGHT=800,480` + BWR 双 plane(`to_planes`/`pack_frame`)。
- `render/grid.py` 从 planes 取尺寸。
- `render/layouts.py` 仅有单一全局 `GRID={cols:8,rows:6}` 与一套 `BUILTIN_LAYOUTS`。
- `render/engine.py` 按写死尺寸建画布、打 BWR 包。
- `render/widgets.py` 多为 Zone 相对坐标 + 按 z.h/z.w 算字号(可缩放),少数用 RED(日历"今天")。
- `server.py` 的 `/frame` 为无状态拉模式,仅读 `t`/`h` 查询参数,帧为全局单一。

### 引入 ScreenProfile

把渲染知识集中到 hub 的一个 profile 表:

```python
bwr_750 = {w:800, h:480, color:"bwr", rotate:0,  grid:(8,6), pack:bwr_2plane}  # 现状
bw_426  = {w:480, h:800, color:"bw",  rotate:90, grid:(4,8), pack:bw_1plane}   # 新增
```

### 改动

- **`planes.py`**:移除写死尺寸/BWR;
  - `pack_frame(img, profile)`:按 `profile.color` 选单 plane(BW)或双 plane(BWR);
    按 `profile.rotate` 在 hub 侧用 PIL 旋转 90°(480×800 → 800×480)后再 1bpp 打包。
  - BW 单 plane:`to_plane_bw(img)` 输出 48000B。
- **`grid.py`**:`cell_to_zone(grid, p, profile)` 用 `profile.w/h`。
- **`layouts.py`**:`BUILTIN_LAYOUTS` 改为**按 profile 分组**:
  - `bwr_750` 沿用现有 8×6 六布局;
  - `bw_426` 新增 4×8 竖版六布局,**同名**(dash/photo/clock/usage/split/todo);
  - `get_layout(path, name, profile)`;用户自建布局存储按 profile 命名空间隔离。
- **`engine.py`**:`render_frame(cfg, state, profile)` 按 profile 建画布、选布局集、渲染、打包。
- **`server.py /frame`**:解析 `&panel=<id>` → 解析 ScreenProfile(**缺省/未知回退 `bwr_750`**,兼容现役固件)→ `render_frame(cfg, state, profile)`;ETag/preview 随 profile。
- **颜色降级**(BW profile):
  - 提供 `accent(profile)`:BWR 返回红、BW 返回黑。
  - 提供 `emphasize(d, zone, ...)`:今天等原本靠红区分的元素改用加框/反色/下划线。
  - 仅日历"今天"、市场涨跌等少数 widget 需改;红→黑后语义靠形状保留。

### 4×8 竖版布局草案(可在 plan 阶段微调)

```
480×800 / 4列×8行(格子 120×100px)
dash:  header(4×1) / claude_status(4×2) / usage(4×2) / todos(4×3)
photo: photo(4×8)
clock: big_clock(4×3) / calendar(4×3) / 余 2 行留白或 header
usage: usage(4×3) / usage_ring(4×2) / claude_status(2×3) + todos(2×3)
split: header(4×1) / claude_status(4×2) / calendar(4×2) / todos(4×3)
todo:  todos(4×5) / calendar(4×3)
```

## 第 4 节 · 兼容性

- 现役 7.5 寸固件不带 `panel` 参数 → hub 回退 `bwr_750` → 行为完全不变(**零回归**)。
- 新 4.2 寸固件带 `&panel=bw_426` → hub 渲 480×800 竖屏 → 旋转 → 48000B 单 plane。

## 第 5 节 · 测试

**hub(可纯软件验证,无需硬件)**:

- profile 解析与 `/frame` 参数解析 + 缺省/未知回退 `bwr_750`。
- 各 profile 网格数学(cell_to_zone 边界无缝)。

- 颜色降级:BW 下红→黑且"今天"有替代强调。
- 6 个竖版布局逐个渲染不抛异常、产出 48000B 帧。
- 扩展现有 `test_planes/test_grid/test_engine/test_layouts` 为 profile 参数化。

**固件**:`ssd1677_selftest` + `INKPULSE_VERIFY` harness;真机校准旋转方向/bit 极性/升压。

## 第 6 节 · 实现拆分(两条独立 plan + 一份硬件清单)

- **Plan A(hub 多 profile)**:profile 重构(planes/grid/engine/layouts)+ bw_426 竖版布局 + 颜色降级 + `/frame` panel 参数 + 测试。
  **可在 hub 用 `/preview.png` 完整验证,不依赖硬件,应先行。**
- **Plan B(固件 ssd1677 驱动)**:Kconfig 选屏 + `ssd1677.c` + caps/URL panel id + BW `mark_offline` + selftest + 真机校准。
- **硬件核对清单**:随 Plan B 收尾(升压电容核对、pin6/7、结构件/外壳重做)。

两者通过 `&panel=` + ScreenProfile 解耦;A 与 B 可并行,A 可独立先验。

## 非目标(YAGNI)

- 不做固件运行时自动探测屏型(编译期 Kconfig 已足够)。
- 不把控制器时序/LUT 参数化下发(不同控制器差异大,得不偿失)。
- 本期不为 4.2 寸引入旧屏没有的新 widget;先复用现有 widget 重排。
