# 仪表盘 WiFi 状态显示设计

日期：2026-06-12
状态：待评审

## 背景与目标

仪表盘由 hub 渲染位图、设备只负责显示。当前从屏上看不出网络状态：离线时屏保留旧帧，用户分不清是 WiFi 断、hub 不可达、还是数据没变。

本功能在 header 右上角设一个**统一的「WiFi 状态指示器」**：在线显示信号强度、离线显示红色告警。

## 设计

右上角同一块区域既是信号指示也是离线告警，两种互斥状态：

### ① 信号图标（在线，hub 渲染）
- **数据流**：设备 `ch_fetch` 用 `esp_wifi_sta_get_rssi()` 取 RSSI，像现有 `t/h` 一样加 URL 参数 `&rssi=<dBm>` 传给 hub。
- hub `/frame` 接收 `rssi`，传入渲染状态；`widgets` 在 header 右上角（温度左侧）按 RSSI 画 WiFi 信号格：
  - `>= -60` → 3 格(强)；`-60~-72` → 2 格；`-72~-82` → 1 格；`< -82` 或无 → 0 格(空心)。
- 颜色：黑色。

### ② 离线告警（离线，设备叠加）
- **触发**：主循环 `ch_fetch` 返回非 `ESP_OK`（http perform 失败 = WiFi 断 / hub 不可达），且**状态翻转**（在线→离线）。
- **形式**：设备在 `s_framebuf` 的**红 plane** 右上角（对齐信号图标区）画一个红色 WiFi 叉图标，`disp->show` + `refresh` 一次。利用 BWR 的红色 plane 做醒目告警。
- **恢复**：再次 fetch 成功时正常 hub 帧覆盖，红叉自然消除。
- **刷新策略**：仅在状态翻转（在线↔离线）时刷一次，避免 e-ink 频繁全刷（一次 21s）。

## 数据流 / 架构

- **RSSI**：固件 → URL 参数 → hub 渲染（在线信号格）。
- **离线叉**：固件本地检测 + 叠加（hub 不参与——离线时本就拉不到帧）。
- **位置对齐**：hub 信号格区 = 固件红叉区（右上角同坐标），保证两种状态视觉连贯。

## 坐标约定

- 指示器区域：header 右上角，约 32×24px；温度数字移到其左侧，避免重叠。
- 固件红叉：把该区域的 EPD 坐标换算成红 plane 字节偏移，写入内置叉位图。

## 错误处理

- `esp_wifi_sta_get_rssi()` 失败：`rssi` 传 0 / 省略，hub 画 0 格，不崩。
- 开机即离线（尚无有效帧）：跳过叠加（屏已 clear），仅靠后续翻转触发。

## 测试

- **hub**（TDD）：`widgets` 信号格按 rssi 画对应格数（渲染不崩、像素断言）；`/frame` 正确接收 `rssi` 参数。
- **固件**：离线叠加靠真机验证（断 hub 看红叉出现、恢复 hub 看红叉消失）。

## 落地顺序

1. hub：`/frame` 收 `rssi` + `widgets` 画信号格（TDD）。
2. 固件：`ch_fetch` 取 RSSI 传 URL 参数。
3. 固件：在线/离线状态翻转检测 + 红叉叠加 + 刷新策略。
4. 真机验证：信号格随 RSSI 变化；断 hub 看红叉；恢复看消失。

## 风险

- header 右上角空间紧张（温度 + 信号格），需调坐标避免重叠。
- e-ink 慢刷：离线红叉刷新占 21s，仅翻转时刷一次可接受。
- 离线叉只在「翻转」时刷；若设备此间被复位，需保证开机首次成功/失败也能正确置位。
